Interface nsINavHistoryResultObserverType

Allows clients to observe what is happening to a result as it updates itself according to history and bookmark system events. Register this observer on a result using nsINavHistoryResult::addObserver.

Properties

Readonly observeHistoryDetails

observeHistoryDetails: boolean

Whether the observer is interested into history details changes. Those include visits additions and removals. If the observer doesn't provide this attribute, it will default to true. In practice, the observer won't receive nodeHistoryDetailsChanged. Note: this is only read when the observer is added, it cannot be changed dynamically.
